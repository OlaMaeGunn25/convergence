Installation & Cloud Architecture Explorer

Deploying a new AI agent instance is simple and automatic. Follow these steps to provision and activate your workspace:

1. SET CONFIGURATION
Enter your company name, color theme, and target vertical. The system locks down parameters automatically.
2. PROVISION ENVIRONMENT
Click 'Activate' to start the installer. The coordinator automatically spawns isolated containers on GCP and migrates database schemas.
3. LIVE API VERIFICATION
The installer checks and verifies system hook connectivity (GCP, Supabase, n8n, Twilio, and mock QuickBooks).
4. AUTO-UNLOCK LAUNCH
Click the Launch button to automatically enter the secure cryptographic vault of your new workspace in one click!
WHAT IS HAPPENING?
Spawning dedicated serverless Docker container runtimes containing the agent's n8n workflow blueprints, Dify.ai cognitive RAG model engine, and isolated Supabase DB schemas.
WHERE IS IT HAPPENING?
Containers execute on **Google Cloud Run (Serverless)** in the `us-central1` zone. Data is stored inside a dedicated postgres schema on **Supabase**. Communications are tunneled via secure **Twilio** hooks.
WHY BUILT THIS WAY?
Isolating workspaces prevents customer data interception (HIPAA/GDPR compliance). Serverless computing ensures zero host idle costs—scaling up dynamically only on inbound tasks.
HOW THE TOKEN WORKS
The token is a signed JWT containing container endpoints & parameters. Passing it to the Operations Hub unlocks the secure KMS Vault, matching prompt vectors & activating the agent runner.
