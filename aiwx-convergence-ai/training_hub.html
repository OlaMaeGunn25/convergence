  • Dedicated Virtual Machine or Docker environment (2 vCPU, 4GB RAM minimum).
  • Active SSL termination mapping to a domain (e.g. assistant.client.com).
  • Direct developer API keys for OpenAI/Claude, QuickBooks, and Twilio.

  3. Generate the Base64 Activation Token and provide it to the client administrator.

Module 5: Scalable Cloud Architecture & Dual-Deployment Models DEPLOYMENT GUIDE

Managed SaaS Model (No Client IT Staff)

For standard SMB clients with zero technical departments, run their container VM securely inside **your own Google Cloud Platform (GCP)** account:

  • Bind a dedicated subdomain (e.g. clientname.convergence-ai-agents.com).
  • Deliver the login URL and their encrypted KMS activation token.
  • Collect high-margin subscription fees covering VM resources and API tokens.

Client-Owned Cloud Model (With Client IT Staff)

For larger clients requiring strict corporate security or HIPAA data compliance on their own clouds (AWS, GCP, Azure, or private server):

  • Provide their IT team with pre-packaged Docker compose manifests.
  • Client manages and pays for their own cloud virtual machine resources.
  • Generate the KMS Activation Token to authorize their self-hosted license.

To protect clients from credential harvesting and data breaches, implement the Secure KMS-Protected Encryption Vault cabinet:

  • No Rebranded "Zero-Trust" Terms: Always refer to the premium security system using the terms "Vault-Grade Cryptographic Shield" or "Secure KMS-Protected Encryption Vault".
Module 9: Google Cloud (GCP) Services & Architecture Provisioning GCP MASTER CLASS

Core Host Setup (Compute & Database)

Configure the high-availability infrastructure required to power your client's administrative agent:

  • Compute Engine (VMs): Provision standard e2-medium or e2-standard-2 Debian virtual machine instances running Docker. Pre-configure auto-restarts for container resilience.
  • Cloud SQL (PostgreSQL): Deploy fully managed Cloud SQL instances running PostgreSQL 15+ with the pgvector extension enabled. This isolates and secures client vector storage.
  • Cloud DNS & Static IPs: Bind standard static external IP addresses and map clean client subdomains (e.g. assistant.clientdomain.com) via Cloud DNS zones.

Security, KMS Vault & Networking

Implement our mandated Vault-Grade Cryptographic Shield guidelines to ensure enterprise-grade security:

  • Cloud KMS Encryption Keys: Create highly secure, symmetric customer-managed encryption keys (CMEK) via Cloud KMS to automatically handle secret decryption inside short-lived container environments.
  • VPC Firewall Hardening: Configure isolated VPC subnets. Create firewall rules restricting SSH access strictly via Identity-Aware Proxy (IAP) tunnels, keeping external ports 22/3389 completely blocked.
  • Cloud Storage (GCS) Backups: Create private GCS buckets with custom object retention policies to house database dumps, document archives, and RAG ingestion files securely.
Module 10: Multi-System Connectors (Bookkeeping & Email Integrations) INTEGRATIONS MASTER CLASS

Flexible Bookkeeping Integration (n8n Nodes)

To connect alternative bookkeeping platforms, utilize self-hosted n8n's modular routing:

  • QuickBooks Online vs. Xero/FreshBooks: The frontend maps to n8n via a standard JSON webhook payload. To swap from QuickBooks to Xero, FreshBooks, or Zoho Books, simply swap the integration node inside n8n's visual workspace.
  • Lightweight Ledgers (Google Sheets/Excel): For solo operators or micro-SMBs, you can configure n8n's native Google Sheets Node to log and parse transactions directly into a dedicated spreadsheet ledger, completely bypassing accounting software seat licensing fees.

Email Client Protocols (Gmail, Outlook & Custom)

Calibrate your client's email intake and dispatch configurations based on their provider:

  • Gmail (Personal @gmail.com): Connect via Google's secure OAuth 2.0 gateway or establish an encrypted SMTP/IMAP protocol connection using a standard 16-character Google App Password (with 2-Step Verification enabled). Daily sending cap is 500 emails.
  • Google Workspace: Authenticate directly through OAuth 2.0. Highly recommended for high-volume enterprises due to standard 2,000 daily sending caps and higher API quotas.
  • Microsoft Outlook / Office 365: Connect natively via MS Graph API OAuth credentials inside n8n or set up encrypted IMAP/SMTP nodes using a secure Outlook App Password.
  • Custom IMAP/SMTP Servers: The agent supports connection to arbitrary self-hosted email servers by inputting secure TLS hostnames, ports, and login keys in n8n's email triggers.
