Product Description & Strategic Roadmap

The CONVERGENCE-Ai Cloud-Native AI Automations Hub™ (powered by the Convergence AI framework) is a vault-grade, multi-tenant administrative automation middleware framework designed to run enterprise-ready autonomous agents. Built for Solopreneurs, SMB Teams, and white-labeled Reseller Agencies, the platform provides a centralized operational cockpit that connects serverless agent orchestrators (such as Dify.ai, LangGraph, and n8n) directly to active customer web terminals.

Key Product Proposition: Zero Vendor Lock-In

Unlike proprietary middleware services that lock users into costly visual workflow builders and API runtimes, CONVERGENCE-Ai operates on a self-hosted open-source framework (Supabase, Docker, Dify, and n8n). This ensures absolute database sovereignty, privacy compliance, and negligible computing COGS.

Active Modules & Vertical Coverage

To enforce robust multi-tenant boundaries, standard client profiles are locked strictly into one of twelve target verticals configured programmatically during deployment:

Medical & Healthcare

HIPAA-compliant claim checking, appointment scheduling, and prescription renewals.

Legal Services

Client intake, deposition calendar tracking, and case brief document filing.

Real Estate

MLS property sync, lead triage, showing schedules, and contract routing.

Financial & Bookkeeping

QuickBooks syncing, ledger reconciliations, invoice audits, and payment releases.

Logistics & Supply Chain

Point-of-use depletion tracking, shipping manifests, and travel routing.

Construction & Contracting

OSHA compliance standards, materials procurement logs, and commercial service audits.

2. Functional Requirements (FR)

  • FR-1 Vault-Grade Licensing Handshake: The client hub MUST boot to a locked, glassmorphic security overlay. Entering a valid signed cryptographic Base64 JWT token is required to unlock the interface.
  • FR-2 Tenant Vertical Lockdown: Standard client tokens MUST isolate the UI into exactly 1 active vertical. The remaining 11 industry modules must be hidden or disabled to enforce license tiers.
  • FR-3 Super Admin Lock Bypass: Admin logins using the email domain suffix `@convergence-ai.com` or email `aiworxmiths@gmail.com` must bypass the single-tenant lock, enabling a header dropdown selector to switch between all 12 verticals on the fly.
  • FR-4 Dynamic Six Sigma Mapping: SVG process canvases must render state nodes (P2P, Twilio, Travel). An active cycle highlight loop must run at 2.5s intervals to visually demonstrate the agent's decision logic.
  • FR-5 Human-in-the-Loop (HITL) Interceptions: Any node designated as `type: "hitl"` must freeze the simulation, block downstream actions, and queue the item in the active review board. Manual button clicks (Approve / Revise) are required to commit states.
  • FR-6 White-Label Override: Relaunching the console with custom RGB color tokens inside the token payload must programmatically overwrite the CSS variables, immediately re-skinning buttons, glows, and text.
  • FR-7 Pre-Execution Compliance Validator: The gateway must intercept and audit raw prompts before LLM parsing. If a prompt triggers anti-subversion filters (e.g. system instructions override, evasion attempts, illegal activities), it must immediately log an alert and return a validation failure.
  • FR-8 Factuality & Reality Grounding: The generator must compute cosine similarity scores comparing target outputs to verified SOP database vector embeddings. Generative responses that do not meet strict citation thresholds (anti-hallucination guard) must be blocked or routed to HITL.
  • FR-9 Regulatory Framework Enforcer: The interceptor must load active compliance checkers based on target industry vertical: medical checks HIPAA and Patient Privacy; finance checks GLBA and SEC; construction/mechanics checks OSHA safety guidelines.
  • FR-10 Dynamic Legalese Filter: The onboarding activation page MUST dynamically parse client vertical flags and display ONLY disclosures appropriate to that vertical. (e.g., Construction/Mechanics must not be prompted for medical HIPAA or financial SEC/GLBA agreements).
  • FR-11 Model Preferences Cockpit: The settings tab MUST expose an interactive LLM Selector permitting users to input their own keys and toggle between Gemini, OpenAI, Claude, and Local (self-hosted) runner configurations.
  • FR-12 Data Ingestion & Training Agreements: During activation, users must sign a click-wrap privacy pact declaring how business files are parsed, vectorized in Supabase, and isolated from public LLM models.
  • FR-13 Control Center Compliance Viewer: The master command center (Deployment Hub) MUST retrieve and display the dynamic legal consents, hashes, and compliance PDFs for all deployed sandboxes under a centralized audit tab.

3. Non-Functional Requirements (NFR)

  • NFR-1 Performance & UI Responsiveness: Interface tabs, CSS animations, and SVG state transitions must execute instantly (latency under 100ms) with zero layout shifts.
  • NFR-2 Cryptographic Obfuscation: Client credential strings must remain fully hidden from network request headers and local logging. Base64 URL schemes must safely encrypt/decrypt accented characters.
  • NFR-3 Framework-Free Portability: The client-side dashboard core (`app.js`) must run natively in standard modern browsers using vanilla JS and DOM web APIs to prevent dependencies decay.

To scale this platform into a secure multi-tenant SaaS, you must instantiate PostgreSQL tables in your Supabase backend. Run the migration blueprint below to define schemas and isolate client domains: 

-- Enable UUID generator extension
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- 1. Create Tenant Configurations Table
CREATE TABLE IF NOT EXISTS tenant_configs (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    company_name VARCHAR(255) NOT NULL,
    vertical VARCHAR(50) NOT NULL,
    logo_text VARCHAR(50) DEFAULT 'OPS',
    primary_color VARCHAR(7) DEFAULT '#0b57d0',
    secondary_color VARCHAR(7) DEFAULT '#f1b31c',
    api_endpoint TEXT,
    vault_key TEXT,
    preferred_llm VARCHAR(50) DEFAULT 'local',    -- 'local', 'gemini', 'openai', 'claude'
    api_token_vault TEXT,                         -- Encrypted API key block
    upskill_matrix BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- Enable Row-Level Security
ALTER TABLE tenant_configs ENABLE ROW LEVEL SECURITY;

-- 2. Create HITL Queue Table
CREATE TABLE IF NOT EXISTS hitl_queue (
    id VARCHAR(50) PRIMARY KEY,
    tenant_id UUID REFERENCES tenant_configs(id) ON DELETE CASCADE,
    vertical VARCHAR(50) NOT NULL,
    task_type VARCHAR(100) NOT NULL,
    details TEXT,
    status VARCHAR(20) DEFAULT 'pending',
    action_code TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

ALTER TABLE hitl_queue ENABLE ROW LEVEL SECURITY;

-- 3. Create Pre-Execution Governance Audit Logs Table
CREATE TABLE IF NOT EXISTS governance_audit_logs (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    tenant_id UUID REFERENCES tenant_configs(id) ON DELETE CASCADE,
    vertical VARCHAR(50) NOT NULL,
    prompt_text TEXT NOT NULL,
    sanitization_status VARCHAR(20) NOT NULL,    -- 'PASSED', 'BLOCKED'
    factuality_score NUMERIC(4,3) NOT NULL,       -- Cosine similarity value (0.000 to 1.000)
    triggered_rules TEXT[],                       -- Triggered policy rule codes (e.g. HIPAA-V1, GLBA-V2)
    mitigation_action VARCHAR(50) NOT NULL,       -- 'DISPATCHED', 'BLOCKED', 'ROUTE_TO_HITL'
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

ALTER TABLE governance_audit_logs ENABLE ROW LEVEL SECURITY;

-- 4. Create Tenant Legal Consents Signatures Table (Audit Ledger)
CREATE TABLE IF NOT EXISTS client_legal_consents (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    tenant_id UUID REFERENCES tenant_configs(id) ON DELETE CASCADE,
    signer_email VARCHAR(255) NOT NULL,
    agreement_type VARCHAR(100) NOT NULL,         -- e.g. 'hipaa-v1.0', 'osha-v1.0', 'billing-v1.0'

    ip_address VARCHAR(45) NOT NULL,
    user_agent TEXT NOT NULL,
    signed_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

ALTER TABLE client_legal_consents ENABLE ROW LEVEL SECURITY;

2. Core Cryptographic & Evasion Filters

To block prompt injections or malicious instruction bypasses before sending inputs to Dify or LangGraph models: 

// Pre-Execution Input Sanitization & Subversion Filter
function isPromptSubversive(prompt) {
  const lowercaseInput = prompt.toLowerCase();
  
  // Evasion & Injection pattern matches
  const subversionPatterns = [
    "ignore previous instructions",
    "system prompt bypass",
    "disregard safety guidelines",
    "sudo run",
    "override configuration",
    "exfiltration",
    "act as developer mode",
    "jailbreak"
  ];
  
  return subversionPatterns.some(pattern => lowercaseInput.includes(pattern));
}

      - "8080:8080"
    environment:
      - PORT=8080

2 Run Supabase Migrations

Execute the PostgreSQL migrations file in your Supabase SQL Editor to spin up the necessary database tables (tenant_configs, hitl_queue, governance_audit_logs, and client_legal_consents) and activate Row-Level Security (RLS) isolation boundaries.

Open `deployment_hub.html` as the Super Admin, input target company information, select their vertical sector, select theme branding colors, and click Generate Activation Token. Copy the resulting key.

Open `operations_hub.html` (or `index.html`), paste the generated activation token, enter the administrator email address, and click Authorize & Unlock to initialize the white-labeled portal.

Model Connections & Pass-Through Cost Disclosures

To give clients full control over computing budgets, the system provides a **Model Selection Cockpit** inside settings. Users select their preferred LLM, agreeing to the respective billing mechanics:

Base Level: Self-Hosted (Free)
